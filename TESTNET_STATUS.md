## Mocks Fixed in This Audit

### 1. `_poseidon2_py()` (SHA-256 → Real BN254 Poseidon) — FIXED ✅

**Before (BROKEN):**
```python
def _poseidon2_py(a, b):
    # SHA-256 — does NOT match Noir's poseidon::bn254::hash_2
    return int(hashlib.sha256(...).hexdigest(), 16) % (2**254)
```

**After (REAL):**
```python
def _poseidon_bn254(a, b):
    # Real BN254 Poseidon — matches Noir's std::hash::poseidon::bn254::hash_2
    return int(poseidon_instance.run_hash([a % BN254_PRIME, b % BN254_PRIME]))
```

**Impact:** Without this fix, nargo prove would ALWAYS fail because Python and  
Noir computed different Merkle roots. The proof constraints `assert(curr == positions_root)`  
would never satisfy.

### 2. Silent Mock Proof Fallback — FIXED ✅

**Before (BROKEN):**
```python
except FileNotFoundError:
    result = SimpleNamespace(returncode=0, stdout="Mocked output", stderr="")
    is_compliant = True  # ← FAKE!
# ...
proof_hex = "0x" + "01" * 32  # ← FAKE PROOF SUBMITTED ON-CHAIN
```

**After (REAL):**
```python
except FileNotFoundError:
    return json.dumps({
        "error": "nargo not found. Install on Linux: noirup",
        "is_compliant": False,
        "proof_hex": "0x00",
    })
```

**Impact:** Previously, fake `0x0101...` proofs were being signed and sent on-chain  
as if they were real cryptographic proofs. Now it fails loudly and honestly.

### Why nargo needs Linux:
- Noir/Barretenberg (nargo) only supports Linux and macOS officially.  
- Windows support is not available as of Nargo v1.x.  
- ZK proof generation (30-120 seconds) requires nargo on the PATH.
